The respoke Object

Introduction

The respoke object is the base object of the Respoke JavaScript client library. A single global instance is created automatically when the library loads. Applications use the respoke object to create instances of the client object and to check the functionality of the browser platform

Properties

version()

Returns: string

Description: Returns the version of the Respoke client library

hasMessaging()

Returns: boolean

Description: Returns true if the browser in which the library is loaded supports Respoke messaging. Returns false if messaging is not supported by the browser.

hasCalling()

Returns: boolean

Description: Returns true if the browser in which the library is loaded supports Respoke audio and video calling. Returns false if calling is not supported by the browser. Note that this does not necessarily indicate that the computer has an operational microphone, speakers or camera.

hasDirectConnection()

Returns: boolean

Description: Returns true if the browser in which the library is loaded supports peer-to-peer data connections. Returns false if direct data connections are not supported by the browser.

hasScreenSharing()

Returns: boolean

Description: Returns true if the browser in which the library is loaded supports screen sharing and if an appropriate browser plugin is installed. Returns false if screen sharing is not supported by the browser.

clients

Data Type: Array

Description: An array representing all active client objects. See the createClient() method for more information on creating client objects.

Methods

createClient(params)

Description: Creates and returns a client object. Unlike the connect() method, this method does not automatically attempt to create a connection. To do so, you will need to use the connect() method of the resulting client object.

Returns: client Object

Arguments: object with named parameters (see 'Parameters' below)

Input Parameters:

Name Data Type Required Description
appId string yes The appId value for the Respoke application with which this client is associated. The is generated by Respoke when the application is created. You can find it in the configuration page for the application in the Respoke Developer Console.
baseURL string no The URL for the Respoke back-end platform to which the Respoke client should connect. ONLY USED FOR ADVANCED DEBUGGING. WE MIGHT WANT TO HIDE THIS.
token string no* The access token provided by the Respoke brokered authentication process. Required if the application is in production mode. Not required if the application is in Development mode.
endpointId string no* The endpoint identifier for the client connection. This is only valid if the client is in Dev Mode. If the client is not in Dev Mode then the endpointId is set using the brokered authentication process.
presence any no A presence value to set for the client connection once the connection is established. See the presence information in the client object for details.
developmentMode boolean no* Indicates whether the application is operating in development mode. WHY DO WE NEED THIS?
reconnect boolean no Should the client automatically attempt to re-connect if the connection drops? Defaults to true. MUST BE SET FALSE FOR PRODUCTION MODE!
enableCallDebugReport boolean no Should the client return debug data to the Respoke platform? Defaults to false.

Callback Parameters:

The following parameters allow you to pass callback functions that are called in response to various client events. Please note that in most cases you could use events or promises.

Name Data Type Event Promise Description
onSuccess function no yes Called when the respoke object sucessfully creates a client object.
onError function no yes Called when the respoke object fails to create a client object. Passes an error object containing information on the error.
onJoin function yes no Called when this client joins a group. Call include a single string parameter containing the name of the group joined.
onLeave function yes no Called when this client leaves a group. Call include a single string parameter containing the name of the group left.
onMessage function yes no Called when a message arrives from a remote endpoint or group. Calls to the callback include a single message object parameter which contains the message data.
onDisconnect function yes no Called when the client disconnects from the Respoke platform.
onReconnect function yes no Called when a previously disconnected client re-establishes a connection to the Respoke platform.
onCall function yes no Called when an incoming call arrives from a remote endpoint. Calls to the callback include a single call object parameter containing details about the call.
onDirectConnection function yes no Called when an incoming direct connection request arrives from a remote endpoint. Calls to the callback function include a single directConnection object parameter.

Example

var connectParams = {
    "appId": "",
    "developmentMode": true,
    "endpointId": "fred.jones@someplace.net",
    "onSuccess": function() {
        console.log("Client object connected.A")
    },
}
var myClient = respoke.createClient(connectParams);
